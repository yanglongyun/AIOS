# AIOS 整体架构

## 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + Vite + Tailwind CSS |
| 后端 | Node.js（ESM）+ better-sqlite3 |
| AI | OpenAI 兼容 API，支持流式与常规调用 |
| 通信 | WebSocket（`ws`） |
| 运行形态 | 本地 Node.js / 宿主机直跑 |

## 服务架构

当前是两个独立 Node.js 服务：

| 服务 | 监听 | 职责 |
|------|------|------|
| `server` | `0.0.0.0:9700` | 对外统一入口，负责静态资源、`/api/*`、`/aios/apps/*` 代理、WebSocket |
| `apps` | `127.0.0.1:9701` | 应用后端运行时，负责各应用 `/apps/*` API |

对外只暴露一个主端口：

| 路径前缀 | 实际处理方 |
|----------|-----------|
| `/aios/api/*` | `server` |
| `/aios/apps/*` | `server` 接收后反向代理到 `apps` |
| `/aios/ws` | `server` |
| `/files/*` | `server` |
| 其他页面资源 | `server` 直接提供 `ui/dist` |

因此，架构是“双服务、单入口”，不是单进程直连所有路径。

## 运行模式

### 本地运行

开发模式：

```bash
npm run dev
```

正式运行：

```bash
npm run build
npm run start
npm run start:apps
```

正式访问入口统一是：

```text
http://localhost:9700/aios/
```

正式部署推荐宿主机直跑，由进程管理器或 `systemd` 分别拉起：

```bash
npm run build
npm run start
npm run start:apps
```

对外仍然只暴露 `9700`，由主服务统一承接前端、API、WebSocket 和 apps 代理。

## Agent 系统

系统 prompt 由以下片段动态拼装：

```text
INSTRUCTION.md → language → memory → environment → model → tools → apps → skills → chats
```

其中 `skills` 目前会同时扫描：

| 来源 | 路径 |
|------|------|
| 项目技能 | `skills/` |
| 系统技能 | `$HOME/.skills/` |

## 任务系统

| 类型 | 入口 | 特点 |
|------|------|------|
| Instant Task | `POST /api/task/create/instant` | 单次 LLM 调用，返回结构化结果 |
| Agent Task | `POST /api/task/create/agent` | 多轮 Agent 执行，可调用工具 |

调度器在 `server/task/schedule/`，负责周期任务与定时任务触发。

## WebSocket

WebSocket 由主服务承载，路径为：

```text
/aios/ws
```

同一连接同一时刻只允许一个 chat 执行流。

## 数据库

| 数据库 | 路径 | 用途 |
|--------|------|------|
| 主数据库 | `database/aios.db` | chats, messages, tasks, schedules, settings, users, sessions |
| 应用数据库 | `database/apps/*.db` | 各应用独立数据 |

## 技能系统

一个技能就是一个目录，核心文件是 `SKILL.md`。可选附带 `scripts/`、`references/`、`assets/`。

设置页中的“技能” Tab 展示的是系统当前可扫描到的技能列表，而不是手写配置。
