# AIOS 整体架构

## 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + Vite + Tailwind CSS |
| 后端 | Node.js（ESM）+ SQLite（node:sqlite 兼容层） |
| AI | OpenAI 兼容 API，流式（SSE）+ 常规调用 |
| 通信 | WebSocket（ws 库） |
| 运行时 | Node.js（本地运行） |

## 服务架构

单进程 Node.js 服务，监听端口 9700，统一处理所有请求：

| 路径前缀 | 职责 |
|----------|------|
| `/api/*` | 系统 API（对话、任务、设置、认证等） |
| `/apps/*` | 各应用 API（直接路由，无代理） |
| `/ws` | WebSocket 连接 |
| `/files/*` | 静态文件下载 |
| 其他 | 前端静态资源（ui/dist） |

应用模块在服务启动时统一加载和初始化，路由按 `match(path)` 分发。没有独立 `9701` 进程。

## 运行模式

前端构建必须在 `项目根目录/ui` 执行：

```bash
cd <项目根目录>/ui && npm run build
```

构建产物输出到 `ui/dist/`，由后端静态资源服务提供。

## Agent 系统

`chat()` 循环：

```
callLlmStream → 有 tool_calls？→ runTools → 继续循环
                     ↓否
                   返回 done
```

当前唯一工具：`shell`。工具结果超出 `toolResultMaxChars` 时截断（头 70% + 尾 30%），最大循环轮次由 `toolMaxRounds` 控制。

## 任务系统

| 类型 | 入口 | 特点 |
|------|------|------|
| Instant Task | `POST /api/task/create/instant` | 单次 LLM 调用，结构化 JSON 输出 |
| Agent Task | `POST /api/task/create/agent` | chat() 循环，支持工具，异步执行 |

调度器每分钟 tick，匹配 `cron` 表达式或 `run_at` 时间点触发。

## WebSocket 协议

| 方向 | 消息类型 | 说明 |
|------|---------|------|
| C → S | `message` | 发送对话消息 |
| C → S | `abort` | 中止当前对话 |
| S → C | `delta` | 流式输出片段 |
| S → C | `tool_call` | 工具调用通知 |
| S → C | `tool_result` | 工具结果通知 |
| S → C | `done` | 对话完成 |
| S → C | `error` | 错误通知 |
| S → C | `tasks_changed` | 任务状态变更 |

同一连接同时只允许一个 chat() 执行。

## System Prompt 构建顺序

```
INSTRUCTION.md → language → memory → environment → model → tools → apps → skills → chats
```

每次对话开始时动态构建，完整内容写入 `debug/` 目录供调试。

## 数据库

| 数据库 | 路径 | 主要表 |
|--------|------|--------|
| 主数据库 | `database/aios.db` | chats, messages, tasks, schedules, settings, users, sessions |
| 应用数据库 | `database/apps/<appname>.db` | 各应用自定义 |
