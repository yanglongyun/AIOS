# 应用创建指导

## 1. 必须创建的文件

后端：

```text
apps/<appname>/
├── APP.md
├── index.js
├── api/index.js
├── service/
└── repository/
```

前端：

```text
ui/src/apps/<appname>/
├── index.vue
└── ...
```

如果该应用需要被系统内其它应用调用，再额外创建：

```text
ui/src/apps/<appname>/protocol.js
```

## 2. APP.md 必须有

每个新应用根目录必须有 `APP.md`，frontmatter 固定如下：

```yaml
---
name: reader
description: 简短描述
backend: apps/reader
database: database/apps/reader.db
---
```

规则：
- `name` 全小写，无空格
- `backend` 写真实目录
- 没有独立数据库就写 `无`

## 3. 后端入口固定格式

`apps/<appname>/index.js` 至少包含：

```js
export default {
  name: 'reader',
  match: (path) => path.startsWith('/apps/reader/'),
  initDb,
  handleApi
};
```

要求：
- 必须有 `name`
- 必须有 `match`
- 必须有 `handleApi`
- 可选 `initDb`
- 可选 `initRuntime`

新应用必须注册到：
- `apps/registry.js`

## 4. API 规则

- 使用完整路径匹配
- 不用动态路由段
- 查询参数统一走 `?id=123`
- `api/index.js` 未命中时返回 `false`

示例：

```js
if (path === '/apps/reader/detail' && req.method === 'GET') {
  const id = Number(url.searchParams.get('id') || 0);
  ...
}

return false;
```

## 5. 数据库规则

- 应用数据库放在 `database/apps/*.db`
- `repository/client.js` 负责连接
- `repository/init.js` 负责建表
- 开发阶段直接删库重建
- 不要设计复杂迁移流程

## 6. 前端注册点

新应用必须注册到：
- `ui/src/apps.js`

示例：

```js
{
  id: 'reader',
  name: '阅读器',
  icon: '📚',
  iconClass: 'icon-reader',
  load: () => import('./apps/reader/index.vue')
}
```

如果支持系统内调用，再加：

```js
protocol: () => import('./apps/reader/protocol.js')
```

## 7. 前端请求路径不能写错

浏览器侧统一走根路径：

- 系统 API：`/api/*`
- 应用 API：`/apps/*`

不要写：

```js
fetch('/api/xxx')
fetch('/apps/xxx')
```

要写：

```js
fetch('/api/xxx')
fetch('/apps/reader/list')
```

## 8. 协议规则

系统内应用调用统一走协议。

统一入口：

```js
import { openProtocol } from '../system/protocol/index.js';
```

统一格式：

```js
await openProtocol({
  app: 'chat',
  action: 'open',
  data: {}
});
```

规则：
- `app`：目标 app id
- `action`：动作名
- `data`：结构化参数

每个应用自己的协议行为，写在自己的 `protocol.js` 中，不要散落在组件里。

## 9. AI 能力调用

需要模型能力时，优先复用共享能力，不要每个应用自己重复造。

常用：
- `agentTaskJson`
- `instantTaskJson`

## 10. 创建完成后必须做的事

前端改动：

```bash
cd ui && npm run build
```

后端改动：
- 不要手工杀进程
- 统一走系统重载接口

## 11. 最小交付清单

- `apps/<appname>/APP.md`
- `apps/<appname>/index.js`
- `apps/<appname>/api/index.js`
- `apps/<appname>/service/*.js`
- `apps/<appname>/repository/client.js`
- `apps/<appname>/repository/init.js`
- `apps/registry.js` 已注册
- `ui/src/apps/<appname>/index.vue`
- `ui/src/apps.js` 已注册
- 如需系统内调用，新增 `ui/src/apps/<appname>/protocol.js`

一句话：
- 目录固定
- 路径固定
- 注册点固定
- 协议入口固定
- 不要自由发挥
