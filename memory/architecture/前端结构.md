# 前端结构

## 目录概览

```text
ui/src/
├── main.ts
├── App.vue
├── router.ts                 Vue Router 入口
├── apps.ts                   应用注册表
├── ws.ts                     WebSocket 客户端
├── auth/                     登录态与安装态
├── data/                     静态配置（如 providers.ts）
├── locale.ts                 语言常量（安装时替换）
├── stores/                   全局状态（toast、windowManager、chatPanel 等）
├── views/                    路由级视图
└── apps/                     各应用前端实现
```

## 路由结构

当前前端不是“无路由”模式，而是明确使用 Vue Router。

核心路由：

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | `DesktopView` | 桌面端主界面 |
| `/mobile` | `MobileView` | 移动端主界面 |
| `/welcome` | `WelcomeView` | 首次安装与初始化 |
| `/login` | `LoginView` | 登录页 |

另有一层“路由到窗口”的映射：
- `/settings` 会打开设置窗口
- `/tasks` 会打开任务中心
- `/files` 会打开文件应用
- 各 app 路径会映射到对应窗口

所以它是“有路由 + 桌面窗口管理”并存的结构。

## 应用注册

应用注册表在：

```text
ui/src/apps.ts
```

每个应用定义：

| 字段 | 作用 |
|------|------|
| `id` | 应用唯一标识 |
| `name` | 显示名称 |
| `icon` | 图标 |
| `load` | 懒加载组件 |
| `defaultSize` | 默认窗口尺寸 |
| `mobileLoad` | 可选移动端组件 |

## 设置页

设置页路径是 `ui/src/apps/settings/`，当前已经拆成多个 Tab：

| Tab | 作用 |
|-----|------|
| 账户 | 修改密码、退出登录 |
| 模型 | provider / apiUrl / apiKey / model |
| 消息 | 上下文轮数 |
| 工具 | 工具截断与轮次限制 |
| 技能 | 显示项目技能与系统技能列表 |
| 通用 | 主题、语言 |

技能 Tab 的数据来自：

```text
GET /aios/api/settings/skills
```

## 多语言

系统没有 i18n 机制，改为安装时文本替换：
- 翻译源文件：`shared/i18n.json`（包含 zh/en 所有 key）
- 源码中使用 `__T_KEY__` 占位符，安装时由 `scripts/apply-locale.ts` 替换为对应语言文案
- 创建应用时，界面文字直接按用户当前语言输出，不需要占位符

## 网络访问约定

浏览器侧统一以 `/aios/` 为 base。

| 前端请求 | 说明 |
|----------|------|
| `/aios/api/*` | 系统 API |
| `/aios/apps/*` | 应用 API |
| `/aios/ws` | WebSocket |

正式模式不依赖 `vite` 代理，真实入口是主服务 `9700`。

## 构建与生效

前端修改后需要重新构建：

```bash
npm run build
```

构建产物输出到：

```text
ui/dist/
```

主服务会直接托管这份产物。
