# AGENT 前端设计语言

AGENT 是本机运行的 ChatGPT 风格对话应用。视觉方向保持温暖、克制、可触摸的拟物风:纸面、柔和边线、浅阴影、橙棕强调。它不是营销站,也不是重后台。

## 核心气质

- 对话是第一体验,界面要轻、稳、好读。
- 侧边栏像 ChatGPT:一级入口清楚,列表紧凑,默认不靠大卡片堆叠。
- 任务、记忆、技能是系统能力入口,但不能混成一个页面概念。
- 小屏默认收起侧边栏,打开后悬浮覆盖。

## 颜色与材质

优先使用 `ui/style.css` 里的 token:

- 背景:暖纸面 `--bg`
- 面板:白色 `--panel`
- 文本:深暖黑 `--ink`
- 次级文字:低对比暖灰 `--muted`
- 强调色:橙棕 `--accent`
- 边线:浅暖灰 `--line`

避免整页变成单一色系。按钮、输入框和列表项要有轻微触感,但不要重阴影。

## 布局

- 主壳在 `ui/App.vue`。
- 侧边栏在 `ui/components/Sidebar.vue`。
- 页面组件放在 `ui/components/`。
- 全局样式在 `ui/style.css`。

固定格式元素要有稳定尺寸,避免 hover 或内容变化导致布局跳动。移动端要检查文字不溢出。

## 对话

- 用户消息、AI 消息、订阅消息、工具调用、工具结果要有明确视觉区分。
- 工具调用和结果成组展示,摘要优先,详情可以展开。
- 附件只显示本地上传后的路径信息,让 AI 通过 shell 读取。
- 代码块和长文本不能撑出页面。

## 任务

- 前端只能查看任务、停止活跃任务、查看任务聊天记录。
- 前端不能提供创建任务入口。
- 任务聊天记录来自 `chats.app = 'task'` 和 `messages`。

## 记忆

- 记忆是用户可管理的数据资产。
- 可见性有 `must`、`star`、`stored`。
- 表单需要清楚编辑标题、描述、正文和可见性。

## 技能

- 技能是本地 `skills/*/SKILL.md` 文件资产。
- 技能页面负责列表和全文阅读。
- 不要把技能写进 memories 表。
