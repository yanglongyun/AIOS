## 移动端 Tab 结构

App.vue 用 `v-show` 控制三个 Tab，保持各自状态：

| Tab | 组件 | 说明 |
|-----|------|------|
| `chat` | `ChatPanelView` | 对话面板，点击历史图标可展开 `ChatView` |
| `apps` | `AppsView` → 各应用组件 | 应用网格，点击图标进入对应 app |
| `tasks` | `TasksView` | 任务列表（进行中 / 历史 / 计划），点击进入 `TaskDetailView` |

## 应用注册

在 `ui/src/apps.js` 的 `appRegistry` 数组中添加一项：

```js
{
  id: 'myapp',           // 唯一标识，与后端 name 一致
  name: '应用名称',      // 显示在应用网格图标下方
  icon: '🆕',            // emoji 图标
  iconClass: 'icon-myapp',
  load: () => import('./apps/myapp/index.vue')  // 懒加载
}
```

注册后应用自动出现在应用网格（AppGrid）中，**不需要配置路由**。

## 应用内二级导航

应用组件通过 `inject('mobileNav')` 获取导航控制函数：

```js
const setMobileNav = inject('mobileNav');

// 进入子页面：设置标题 + 返回回调
setMobileNav('子页面标题', () => {
  // 返回逻辑
  setMobileNav(null, null);
});
```

TopBar 会根据 `mobileNav` 状态显示返回按钮和标题。

## 关键约定

- **无路由**：应用不需要配置 Vue Router，由 App.vue 统一切换
- **无 i18n**：所有界面文字直接写中文字面量
- **构建后生效**：修改前端代码后必须执行 `cd <项目根目录>/ui && npm run build` 才能看到效果
- **不要猜目录**：不要在项目根、`/workspace`、`ui/src` 或其他目录直接执行 `npm run build`
- **懒加载**：appRegistry 中的 `load()` 是动态 import，首次打开应用时才加载组件
