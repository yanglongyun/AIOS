## 关键关系

### 主服务 `server/`

- 对外唯一入口
- 托管 `ui/dist`
- 处理 `/api/*`
- 处理 `/aios/ws`
- 把 `/aios/apps/*` 反向代理到 `apps` 服务

### 应用服务 `apps/`

- 独立 Node 进程
- 只监听本机回环地址
- 每个应用通过 `match(path)` 匹配自己的 API
- 由主服务统一暴露给前端

### 前端 `ui/`

- 使用 Vue Router
- 路由负责登录、欢迎页、桌面/移动端入口
- 桌面应用通过窗口管理器打开，不是每个应用都单独占一个顶级页面

### 技能 `skills/`

- 项目本地技能目录
- 每个子目录至少包含一个 `SKILL.md`
- 会和 `$HOME/.skills/` 一起参与技能扫描

## API 分层

| 前缀 | 说明 |
|------|------|
| `/api/*` | 主服务系统能力 |
| `/apps/*` | 应用后端能力，由 `apps` 服务处理 |
| `/files/*` | 文件下载 |

前端实际访问时统一走：

| 浏览器入口 | 实际去向 |
|------------|---------|
| `/aios/api/*` | `server` |
| `/aios/apps/*` | `server` → 代理到 `apps` |
