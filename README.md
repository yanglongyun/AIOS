# AIOS

**AI-Powered Operating System**

---

## 对话，是新的人机语言

AIOS，即由 AI 驱动的操作系统。在 AIOS 中，你可以用最自然的方式向计算机发出指令——对话。

对话是个性化的，是高效的。你可以在一段连续的对话中表达多个需求，它拥有上下文，让你的每一次操作都富有连贯性。你不需要在不同应用之间反复切换，不需要记忆快捷键，不需要学习每个软件的操作逻辑——你只需要说出你想要什么。

毫无疑问，对话正在成为一种革命性的人机交互范式，重新定义了人与计算机之间的关系。它让计算机第一次真正开始理解人，而不是让人去适应计算机。

---

## 但图形界面不会消失

对话自然，却并不总是足够。

语言是抽象的，有时甚至是费力的。当你试图通过文字描述一个复杂的界面布局，或者精确指定某个颜色、某种排版，语言反而成了阻碍。更重要的是，对话是线性的、流动的——它擅长表达意图，却不擅长承载结果。

想象一个只有聊天窗口的系统。当你想记录一篇笔记，它会被淹没在滚动的对话记录里：没有归集，没有文件夹，无法组织，难以搜索，不能置顶，也无从排版。没有"笔记"这个具体的载体，你不会真正去记录笔记。因为你需要的是一个文件，而不是一条回复。

日历、账本、阅读器、画板——每一个具体的需求背后，都需要一个与之匹配的具体形态。形态不是装饰，形态即是功能。

**未来不会流于无形，依然会留于物形。**

对话与界面，从来不是竞争关系。对话负责理解你，界面负责承载结果。两者相辅相成，缺一不可。我们相信，在真正的 AI 操作系统里，图形界面依然是不可或缺的核心。

## 每个人都可以拥有专属的软件

AI 带来的另一场革命，是创作门槛的彻底消失。

在当下这个节点，几乎所有代码都将由 AI 完成。这意味着你完全可以根据自己的需求、习惯与喜好，打造专属于你的软件——你不需要学习任何编程语言，不需要雇佣开发者，不需要等待某个产品团队把你的需求排进路线图。你只需要向 AI 说出你的想法。

在传统软件世界里，大多数人只能接受由开发者定义的产品：接受它的设计语言，接受它的功能边界，接受它认为你"应该"怎么使用它。软件是开发者的软件，不是你的软件。

而现在，这一切正在改变。软件第一次真正可以属于使用它的人。

这些由你与 AI 共同打造的应用，按照你的想法持续演进。你可以使用它，AI 也可以使用它。应用成为你与 AI 协作的平台，是 AI 更好理解你的窗口，也让每一次交互都拥有更丰富、更私人的上下文。你的应用记得你的偏好，懂得你的习惯，专门为你而生。

---

## 应用本身，也在被 AI 重新定义

传统应用中，功能由固定的程序代码决定，边界清晰，能力有限。开发者写了什么，用户就能用什么，不多也不少。

而在 AIOS 中，应用可以直接向 AI 发送任务。这意味着应用不再受限于预先编写的功能逻辑——它可以随时调用 AI 的通用能力，具备更大的灵活性与更强的执行力。一个记账应用可以在你输入完账单后自动生成分析报告；一个阅读器可以在你划线时立即展开深度讨论；一个日历可以主动理解你的日程压力，替你重新规划。程序背后，不需要复杂的编排，AI 即是那个随时待命的引擎。

---

## 一个工具，实现一切

许多 AI 系统试图通过堆叠复杂性来解决问题——多 Agent、多工具、技能系统、记忆模块、MCP 协议……每增加一层抽象，系统就多一份脆弱，多一份难以掌控。

AIOS 选择了相反的路径。

**在 AIOS 中，只有一个工具：指令。**

用指令控制计算机。就这一件事，但这一件事可以实现一切。

你想要记忆？让它建立一个文件夹，把需要存储的内容放进去。下次，它知道去哪里找。

你想要扩展能力？它可以修改自己的代码，刷新页面——然后，它就拥有了那个能力。你想要 MCP 支持？你想要技能系统？你想要工具集成？告诉它就够了。

你想实现上下文压缩，让对话更高效？告诉它。
你想要每次回答后自动给出后续建议？告诉它。
你偏爱温馨的粉色主题，想要聊天背景换成你喜欢的壁纸？告诉它。
你想要连接 Telegram 机器人、Discord 频道，让它在任何地方都能找到你？都没有问题。

它可以帮你撰写研究报告，生成演示文稿，整理资料，起草邮件，分析数据——不是通过调用某个预设的技能，而是真正理解你的需求，然后去完成它。

简单的底层，涌现出无限的可能。这才是我们认为 AI 系统应该有的样子。

---

## 这便是 AIOS

以上，是 AIOS 的核心出发点：对话与界面并存，AI 赋能每个人创造自己的软件，用最简单的底层原语实现最大的灵活性。

基于此，我们构建了一个 Web 系统，并在其中内置了一系列精心设计的应用。这些应用或许并不完全符合你的需求——这完全没有关系。它们存在的意义，是让你看到可能性，并成为你改造的起点。

**AIOS 不是一个等待你适应的系统，而是一个为你而生、由你定义、随你生长的系统。**

## 快速开始

```bash
git clone https://github.com/valueriver/aios.git
cd aios
npm install
npm run build
npm start          # 主服务: 9700
npm run start:apps # 应用服务: 9701
```

打开：`http://localhost:9700`

## 技术栈

- **前端**：Vue 3 + Vite + Tailwind CSS
- **后端**：Node.js + better-sqlite3
- **AI**：Claude API（OpenAI 兼容格式）
- **架构**：三层分离（api → service → repository）

## 开发命令

```bash
npm run dev          # server + apps + ui 同时启动
npm run build        # 构建前端
npm start            # 启动主服务
npm run start:apps   # 启动应用服务
```
