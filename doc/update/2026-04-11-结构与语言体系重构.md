# 2026-04-11 结构与语言体系重构

## 本次目标

这次重构的目标是把 AIOS 的目录语义彻底理顺，去掉长期混乱的中间态。

重点包括：

- 主后端、应用后端、共享代码、Agent、LLM、Prompt 分层
- 顶层 `apps/` 不再承载应用服务代码
- `ui/` 更名为 `gui/`
- `apps/` 与 `memory/` 改为语言驱动的生成目录
- `AIOS/` 主仓库只作为源码源，不再承担运行与调试职责

## 目录结构调整

当前后端结构：

```text
server/
├── main/
├── apps/
├── shared/
├── agent/
├── llm/
└── prompt/
```

语义如下：

- `server/main/`：主系统后端
- `server/apps/`：应用后端
- `server/shared/`：共享后端代码
- `server/agent/`：Agent 层
- `server/llm/`：模型层
- `server/prompt/`：提示词层

这次重构同时删除了旧的兼容入口，不再保留“先兼容再迁移”的壳结构。

## 前端目录调整

前端目录从：

```text
ui/
```

改为：

```text
gui/
```

相关影响：

- `package.json` 的 `dev` / `build` 脚本改为指向 `gui/`
- `server/main/system/language.js` 改为替换 `gui/src`
- `server/main/system/http.js` 改为服务 `gui/dist`
- 所有应用说明、开发文档、提示词文档统一改为 `gui/src/...`

## apps 目录语义调整

顶层 `apps/` 不再是源码目录。

现在的规则是：

- 应用后端代码放在 `server/apps/<appname>/`
- 应用前端代码放在 `gui/src/apps/<appname>/`
- 应用说明文档放在 `language/<lang>/apps/<appname>/APP.md`
- 顶层 `apps/` 只在语言应用阶段生成

这意味着：

- 主仓库默认可以没有顶层 `apps/`
- `server/prompt/apps.js` 读取的是运行态生成出来的 `apps/`
- 只要在进入可运行源码状态前执行过语言应用，顶层 `apps/` 就会出现

系统级应用仍然保留应用说明，但其后端不在 `server/apps/`，而在主系统层：

- `chat`
- `settings`
- `tasks`

## memory 目录语义调整

顶层 `memory/` 也不再是源码目录。

现在的规则是：

- 记忆文档语言源放在 `language/<lang>/memory/`
- 顶层 `memory/` 只在语言应用阶段生成
- `server/prompt/index.js` 读取的是运行态 `memory/index.md`

这意味着：

- 主仓库默认可以没有顶层 `memory/`
- 只有在语言应用后，运行态 `memory/` 才存在

## language 目录重组

语言目录现在按两类内容拆分：

```text
language/
├── zh/
│   ├── gui/
│   ├── apps/
│   └── memory/
└── en/
    ├── gui/
    ├── apps/
    └── memory/
```

语义如下：

- `language/<lang>/gui/`：前端源码替换源
- `language/<lang>/apps/`：应用说明文档源
- `language/<lang>/memory/`：记忆文档源

`applyLanguage(lang)` 现在会同时做三件事：

1. 替换 `gui/src` 中的语言文本
2. 生成顶层 `apps/`
3. 生成顶层 `memory/`

## 开发与运行语义调整

当前约定：

- `AIOS/` 主仓库是源码源
- 不在 `AIOS/` 主仓库里执行 `npm install`
- 不在 `AIOS/` 主仓库里直接调试运行
- 运行验证统一在 `AIOS-dev` 中进行

`AIOS-dev` 当前承担：

- 同步主仓库源码
- 应用语言
- 生成运行态 `apps/` 和 `memory/`
- 安装依赖并启动开发环境

## 文档同步

这次重构后，以下文档已经按新结构重写或更新：

- `doc/prompt/INSTRUCTION.md`
- `server/prompt/INSTRUCTION.md`
- `language/<lang>/memory/app-creation-guide.md`
- `language/<lang>/apps/*/APP.md`

其中：

- `APP.md` 已统一压缩篇幅，只保留定位、路径、数据和入口
- 应用创建指导已经基于 `server/apps + gui/src/apps + language/<lang>/apps` 重写

## 验证情况

已在 `AIOS-dev` 验证通过：

- 主仓库默认不存在顶层 `apps/`
- 主仓库默认不存在顶层 `memory/`
- 运行 `node scripts/refresh.mjs` 后，会自动生成运行态 `apps/` 与 `memory/`
- `http://127.0.0.1:9500/api/health` 返回成功
- `http://127.0.0.1:9501/apps/health` 返回成功

## 当前结论

这次重构完成后，AIOS 的源码结构、语言结构、运行态结构已经分开：

- 源码层：`server/`、`gui/`、`language/`
- 运行态生成层：`apps/`、`memory/`
- 测试运行层：`AIOS-dev`

后续如果继续扩展：

- 应用说明走 `language/<lang>/apps/`
- 记忆文档走 `language/<lang>/memory/`
- 前端语言走 `language/<lang>/gui/`
- 不再回到“顶层 `apps/` / `memory/` 既是源码又是运行态”的旧模式
